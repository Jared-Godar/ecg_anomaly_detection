This project began as a 2022 educational ECG classification experiment and has been rebuilt
from scratch into a governed, tested, reproducible data-engineering showcase — developed
through model-agnostic agentic coding under a tracked operating contract. v1.1.0 is the second
stable snapshot of that system: the release where the governance framework stopped being
documentation and became running automation.

**Important use limitation:** this is not medical software. It is for research, education, and
software-engineering demonstration only — never for diagnosis, monitoring, treatment, or
patient-care decisions. It has not been clinically validated, and this release does not imply
production readiness, clinical suitability, medical utility, or regulatory compliance. No
dataset, derived data, or trained model is distributed here.

## Greatest hits since v1.0.0

### Agentic Engineering

- **Standing commitments, in the repo, not in agent memory** — `AGENTS.md` now carries hard
  contracts that bind every coding-agent session, including cold-start cloud sessions:
  session-handoff continuity (wind-downs produce a state snapshot with copy-pasteable next
  steps), self-recording promises (a rule that lives only in conversation is not considered
  made), and a defensive-external-calls engineering discipline.
- **Durable multi-agent context** — tracked Kiro IDE steering files (project context, workflow
  rules, board conventions, seed-prompt template) give any agent, in any tool, the same
  operating picture; machine-specific settings stay deliberately untracked.
- **The workflow audits itself** — agent-facing rules are enforced by the same CI gates that
  bind humans, so the contract is testable rather than aspirational.

### Governance

- **Creation-time board automation** — new issues and PRs are added to the project board and
  their label-derived fields populated automatically, with curated values always winning over
  the automation; a weekly read-only drift detector backstops the whole board.
- **The changelog is a merge gate** — a dedicated CI job now rejects any substantive PR that
  does not update `CHANGELOG.md`; release-time archaeology is structurally impossible.
- **A stricter, more expressive metadata gate** — nine required project fields, assignee,
  labels, milestone, and closing references validated on every PR, now including a sanctioned
  non-closing issue-reference marker for referencing without auto-closing.
- **Taxonomy stewardship** — a completed legacy-label migration, two new portfolio signals
  (testing-rigor, agentic-engineering), merge-time board-status sync, and tightened branch
  protection (squash-only merges).
- **Late-cycle contract hardening** — the session-handoff continuity contract now codifies a
  worktree-safe fallback chain, so pruning a worktree can never delete the only copy of a
  handoff (#236), and the label taxonomy was aligned with the project board's field options,
  including a newly minted `portfolio: governance` label (#237).

### Reproducibility

- **A fully auditable governed evaluation record** — the one protected-test execution this
  release completed (see the evaluation posture below) is documented with the complete
  evidence chain: clean commit hash, config digests, split identity, approval and rerun
  digests, runtime, and hardware ([record](../held-out-evaluation.md)).
- **Graceful lineage capture** — run manifests now degrade gracefully when git state is
  unavailable instead of failing the run, keeping lineage best-effort-honest rather than
  brittle.
- **Clean-checkout verification, re-earned for this release** — on 2026-07-16, a fresh
  `git clone` into a throwaway location at the final pre-tag commit, followed by
  `uv sync --locked` across the dev, notebooks, and experiments groups (CPython 3.12.13,
  142 locked packages), the full test suite (`uv run pytest -q`: 617 passed), and
  `uv run pyright` (0 errors, 0 warnings), all completed cleanly — the equivalent v1.0.0
  claim was re-verified for this tag, not inherited.

### Testing Rigor

- **An enforced internal-documentation standard** — every supported Python file and the
  curated notebooks carry audited docstrings and commentary, with a merge-blocking gate
  script; the standard was applied repo-wide, not just to new code.
- **The suite grew with the system** — 617 tests across unit, integration, and script tiers
  under strict pytest configuration, including direct unit coverage for previously
  integration-only evaluation arithmetic and for the new governance automation's mapping
  invariants, precedence, and idempotency.
- **Notebooks are tested software** — all three curated public notebooks execute end-to-end
  in CI against synthetic data, with hardened connectivity paths and honest recovery guidance.

### Operational Maturity

- **Defensive external calls, codified and implemented** — dataset acquisition retries
  transient failures (timeouts, resets, `IncompleteRead`, transient 5xx) with bounded
  exponential backoff, fails fast on permanent errors, and on exhaustion names what failed
  and how to recover — no raw tracebacks on user-facing surfaces.
- **Qualified progress everywhere** — per-stage banners with elapsed and remaining-duration
  estimates across the pipeline and all thirteen standalone CLI stages, flushed for live
  streaming without per-iteration noise.
- **Quota-aware governance tooling** — the GitHub automation layer classifies primary vs.
  secondary rate limits, retries with backoff, and budgets its own GraphQL spend.

### Documentation & Presentation

- **A recruiter-facing README** — reframed as an engineering showcase whose headings are the
  portfolio signals themselves; the modernization story is the setup, not the headline.
- **Version-controlled diagrams** — Graphviz `.dot` sources replaced ASCII art as the
  repository's diagram toolchain.
- **Signals at a glance** — CI, license, and Python-version badges; banner artwork; and a
  Development Status advance from Alpha to **Beta**.
- **Deep-dive portfolio pages ship with the release** — six curated `docs/portfolio/` pages
  expand the README's portfolio-signal sections into evidence-linked readouts, joined by
  in-repo release-notes pages for v1.0.0 (retrospective) and this release (#245), plus a
  maintainer editorial pass adding a first-person "Why this project" section to the README
  (#246).

## Honest evaluation posture

v1.0.0 shipped with a deliberate boundary: the protected test partition had never been opened,
and completing that arc was explicitly deferred rather than rushed against a release deadline.
This release closed it the governed way — the held-out protocol defined in
[`docs/benchmark-governance.md`](../benchmark-governance.md) was executed **exactly once**,
against a frozen candidate, with maintainer approval recorded before any protected data was
opened and an append-only rerun history. The bounded result and its limitations are recorded,
without embellishment, in [`docs/held-out-evaluation.md`](../held-out-evaluation.md).
Supported evaluation remains validation-only; the test partition stays closed. The archived
2022 metrics remain historical evidence only
([audit](../historical-results.md)) — no historical or validation-only number is presented
as a benchmark, here or anywhere in the repository.
